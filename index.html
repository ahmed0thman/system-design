Groq + Llama 4 Scout Vision — handles food photo analysis. Free tier supports high volume. Model: meta-llama/llama-4-scout-17b-16e-instruct
Meal Photo Scan — Full Logic Flow
```
STEP 01
Flutter → Gateway
User taps camera,
takes food photo
SEND
POST /api/v1/meals/scan
Authorization: Bearer <jwt>
Content-Type: multipart/form-data
body: { image: File, portion_hint?: string }
STEP 02
Gateway Validates
Auth guard checks JWT,
class-validator runs,
multer handles file
CHECK
JWT verified → userId extracted
File validated → mime: image/*, maxSize: 10MB
Upload to R2 → meals/{userId}/{uuid}.jpg
FAIL
401 Unauthorized — invalid/expired token
413 Payload Too Large — file > 10MB
422 Unprocessable — not an image
STEP 03
Queue Job Created
Async processing starts,
user gets instant jobId
ENQUEUE
BullMQ → queue: "meal-scan"
payload: { jobId, userId, imageUrl, portionHint }
RECV
HTTP 202 Accepted
{ jobId: "uuid", status: "processing", estimatedMs: 2000 }
STEP 04
Groq Vision Call
Worker picks job,
calls Llama 4 Scout
with image + prompt
SEND → GROQ
POST https://api.groq.com/openai/v1/chat/completions
model: "meta-llama/llama-4-scout-17b-16e-instruct"
messages[0]: { role: "user", content: [image_url, text_prompt] }
prompt: "Analyze food. Return JSON: {foods[], calories, protein, carbs, fat}"
RECV ← GROQ
{ foods: [{name:"chicken breast", grams:150, calories:248}], totalCalories: 412, protein: 38, carbs: 22, fat: 12 }
STEP 05
Zod Validates AI
AI response parsed +
validated before
touching DB
VALIDATE
drizzle-zod schema → insertMealSchema.parse(aiResponse)
Checks: calories ≥ 0, protein ≥ 0, foods.length > 0
FAIL
ZodError → job marked failed, user notified to retry
e.g. calories: -1 → fallback: ask user to confirm manually
STEP 06
Save to PostgreSQL
Validated meal written
to DB via Drizzle
INSERT
table: meals + meal_items
{ id, userId, imageUrl, calories:412, protein:38, carbs:22, fat:12, foods:[...], scannedAt }
RECV
meal.id = "uuid" → job marked completed
Daily totals recalculated in nutrition table
STEP 07
WebSocket Push
Flutter app receives
result live, no polling
EMIT → WS
event: "meal.scan.complete"
room: user:{userId}
{ jobId, meal: { id, calories, protein, carbs, fat, foods[] }, dailyTotal }
FLUTTER RECV
Riverpod notifier updates state → UI rebuilds
Meal card appears in today's log ✓
Calories ring animates to new total ✓
```

Llama 4 Scout via Groq — streaming text responses for coach conversations. User meal history injected as context for personalized advice.
AI Coach Conversation — Full Logic Flow
```
STEP 01
Flutter → Coach API
User sends message
to AI coach
SEND
POST /api/v1/coach/message
body: { sessionId?: string, message: "Why am I not losing weight?" }
STEP 02
Load Context
Fetch user history
to inject into prompt
QUERY PG
Last 7 days meals → avg calories, macros, meal times
User profile → age, weight, height, goal (lose/gain/maintain)
QUERY MONGO
coach_sessions → last 10 messages for conversation continuity
coach_context → user memory (known preferences, past insights)
BUILT PROMPT
System: "You are a nutrition coach. User data: {profile}. Recent meals: {mealSummary}. Past conversation: {history}"
STEP 03
Groq Streaming
Llama 4 Scout generates
response, streamed
back via SSE
SEND → GROQ
POST https://api.groq.com/openai/v1/chat/completions
model: "meta-llama/llama-4-scout-17b-16e-instruct"
stream: true
messages: [system_ctx, ...history, user_message]
STREAM ← GROQ
SSE chunks: data: {"delta":{"content":"Based on your meals..."}}
Each chunk forwarded immediately to Flutter via SSE response
STEP 04
Persist Session
After stream ends,
save to MongoDB
INSERT MONGO
coach_sessions → { sessionId, userId, role:"user", content, timestamp }
coach_sessions → { sessionId, userId, role:"assistant", content: fullResponse, timestamp }
STEP 05
Flutter Renders
Streaming text appears
word by word in UI
FLUTTER RECV
StreamBuilder listens to SSE stream
Text widget appends each chunk → typewriter effect ✓
On stream end → show action buttons (log meal, set goal)
```
Real-Time Chat — Full Logic Flow
```
STEP 01
WS Handshake
Flutter connects on
app launch
CONNECT
wss://api.yourdomain.com/socket.io/?token=<jwt>
NestJS WsGuard verifies token → attaches userId to socket
SERVER ACK
event: "connected" → { socketId, userId }
Auto-join rooms: user:{userId}, all active conversation rooms
STEP 02
Send Message
User types and sends
in a chat room
EMIT → SERVER
event: "chat.message"
{ conversationId: "uuid", content: "Great progress today!", type: "text" }
STEP 03
Server Processes
NestJS WS gateway
handles event
VALIDATE
class-validator → content not empty, type in enum
Auth check → sender is member of conversationId
INSERT MONGO
collection: messages
{ _id, conversationId, senderId, content, type, sentAt, readBy:[] }
STEP 04
Broadcast
Message pushed to
all room members
EMIT → ROOM
room: conversation:{conversationId}
event: "chat.new_message"
{ messageId, senderId, senderName, content, sentAt }
OFFLINE USERS
BullMQ → enqueue push notification job
FCM → "New message from {senderName}"
FLUTTER RECV
Socket listener → message appended to chat list
Read receipt → emit "chat.read" → updates readBy[] ✓
```
All System Interactions — Send / Receive Matrix
FROM TO METHOD SENDS RECEIVES
Flutter NestJS Gateway HTTP POST multipart { image: File, portionHint? } { jobId, status: "processing" } 202
Flutter NestJS Gateway HTTP POST JSON { email, password } { accessToken, refreshToken, user }
Flutter NestJS WS WebSocket connect token in handshake query { socketId } — joins user room
Flutter NestJS WS emit "chat.message" { conversationId, content, type } broadcast to room members
Flutter NestJS SSE HTTP GET stream { sessionId, message } SSE text/event-stream chunks
NestJS Gateway Auth Module Guard intercept JWT from Authorization header { userId, role } or 401
Meals Module R2 / S3 HTTP PUT (presigned) image binary stored URL
Meals Module BullMQ queue.add() { jobId, userId, imageUrl } job queued, worker picks up async
BullMQ Worker Groq API HTTP POST JSON { model, messages:[{image_url, prompt}] } { foods[], calories, protein, carbs, fat }
BullMQ Worker Zod (drizzle-zod) schema.parse() raw Groq JSON response typed MealInsert | ZodError
Meals Module PostgreSQL Drizzle INSERT validated meal + items rows meal.id (uuid)
Meals Module WS Gateway server.to(room).emit() { meal, dailyTotal } Flutter receives "meal.scan.complete"
AI Coach Module PostgreSQL Drizzle SELECT userId, last 7 days meal history, user profile
AI Coach Module MongoDB Mongoose find() { sessionId, limit: 10 } last N messages for context
AI Coach Module Groq API HTTP POST stream:true { model, messages:[sys+history+user], stream:true } SSE chunks → forwarded to Flutter
Chat Module MongoDB Mongoose insertOne() message document { _id, sentAt }
Chat Module Redis Pub/Sub publish(channel, msg) serialized message all WS instances receive (horizontal scale)
Notifications Module BullMQ queue.add("push") { userId, title, body, data } job processed by notification worker
BullMQ Worker FCM HTTP POST JSON { token, notification, data } { success: true } or error
NestJS Redis SET / GET / INCR session keys, rate limit counters cached values, TTL-based expiry
